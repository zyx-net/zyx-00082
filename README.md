# 社区托育临时接送授权系统

一个用于处理儿童临时由非日常接送人带走场景的安全管理系统，包含完整的授权申请、审批、核验、交接流程和审计追踪。

## 功能特性

### 核心流程
- **申请授权**：监护人提交接送时间窗和接送人信息
- **审批授权**：另一位监护人审批授权申请
- **核验证件**：老师核对接送人证件尾号
- **完成交接**：核验通过后完成儿童交接
- **撤销授权**：申请人可撤销未完成的授权
- **自动过期**：超过时间窗的授权自动失效
- **拒绝申请**：审批人可拒绝申请并说明原因

### 安全校验
- ✅ 证件尾号格式校验（必须4位数字）
- ✅ 证件尾号匹配校验（与申请时登记的比对）
- ✅ 时间窗有效性校验（不能过期、时长不超过24小时）
- ✅ 审批权限校验（必须是另一位监护人，不能自审自批）
- ✅ 同日冲突校验（同一儿童同一日期只能有一个有效授权）
- ✅ 角色权限校验（不同角色操作不同功能）

### 审计与导出
- 📝 完整的操作轨迹记录（每一步操作都有日志）
- 📊 仪表盘统计（各状态数量统计）
- 📤 导出当日交接清单（CSV格式）
- 🔍 审计日志查询（管理员专属）

## 技术栈

- **前端**：React 18 + TypeScript + Vite + Tailwind CSS + Zustand + Lucide React
- **后端**：Express + TypeScript + better-sqlite3
- **数据库**：SQLite（嵌入式，无需额外安装）

## 主流程演示

### 完整流程步骤

1. **监护人A（如 guardian1 张伟）提交申请**
   - 登录后进入"申请授权"页面
   - 选择儿童（如小明）
   - 填写接送人信息：姓名、电话、证件尾号（如 1234）、与儿童关系
   - 选择接送时间窗（开始时间和结束时间）
   - 提交申请

2. **监护人B（如 guardian2 李娜）审批**
   - 登录后进入"待我审批"页面
   - 看到待审批的申请
   - 点击"通过"或"拒绝"（拒绝需填写原因）
   - 审批通过后，授权状态变为"已批准"

3. **老师核验证件**
   - 使用 teacher1 登录
   - 进入"核验交接"页面
   - 看到待核验的授权
   - 点击"核验"按钮
   - 输入接送人出示的证件尾号
   - ✅ 如果尾号匹配（1234），核验通过，状态变为"已核验"
   - ❌ 如果尾号不匹配（如输入 0000），提示错误并记录核验失败日志

4. **完成交接**
   - 核验通过后，卡片显示"完成交接"按钮
   - 点击"完成交接"
   - 状态变为"已完成"，交接成功

### 异常场景演示

1. **非法证件尾号拦截**
   - 在核验时输入非4位数字或错误尾号
   - 系统提示具体错误原因
   - 核验失败会被记录到审计日志

2. **过期时间窗拦截**
   - 创建一个时间窗已过的授权（或等待时间窗过期）
   - 系统自动将状态置为"已过期"
   - 无法进行核验操作

3. **无权限审批拦截**
   - 申请人尝试审批自己的申请
   - 或非监护人尝试审批
   - 系统提示权限错误

4. **同日冲突授权拦截**
   - 为同一儿童在同一日期创建第二个授权
   - 系统提示冲突错误，不允许创建

## 数据持久化说明

- 所有数据存储在 `data/childcare.db` SQLite 数据库文件中
- 重启应用后，所有数据保持不变：
  - ✅ 待核验事项
  - ✅ 历史轨迹（审计日志）
  - ✅ 导出清单
  - ✅ 页面统计数据
- 如需重置数据，删除 `data/childcare.db` 文件后重启应用即可

## API 接口

### 认证接口
- `POST /api/auth/login` - 登录
- `POST /api/auth/logout` - 登出
- `GET /api/auth/me` - 获取当前用户信息

### 儿童接口
- `GET /api/children` - 获取儿童列表
- `GET /api/children/:id` - 获取儿童详情

### 授权接口
- `GET /api/authorizations` - 获取授权列表
- `GET /api/authorizations/pending-approval` - 获取待审批列表
- `GET /api/authorizations/pending-verification` - 获取待核验列表
- `GET /api/authorizations/:id` - 获取授权详情
- `POST /api/authorizations` - 创建授权申请
- `POST /api/authorizations/:id/approve` - 审批通过
- `POST /api/authorizations/:id/reject` - 拒绝申请
- `POST /api/authorizations/:id/verify` - 核验证件
- `POST /api/authorizations/:id/complete` - 完成交接
- `POST /api/authorizations/:id/cancel` - 撤销授权
- `GET /api/authorizations/:id/audit-logs` - 获取审计日志

### 管理员接口
- `GET /api/admin/statistics` - 获取统计数据
- `GET /api/admin/export/today` - 导出当日清单
- `GET /api/admin/audit-logs` - 获取所有审计日志

## 状态说明

| 状态 | 说明 |
|------|------|
| `PENDING_APPROVAL` | 待审批 - 已提交申请，等待另一位监护人审批 |
| `APPROVED` | 已批准 - 审批通过，等待老师核验 |
| `REJECTED` | 已拒绝 - 审批被拒绝 |
| `PENDING_VERIFICATION` | 待核验 - 等待老师核验 |
| `VERIFIED` | 已核验 - 证件核验通过，等待交接 |
| `COMPLETED` | 已完成 - 交接完成 |
| `CANCELLED` | 已撤销 - 申请人撤销了授权 |
| `EXPIRED` | 已过期 - 超过时间窗自动失效 |

## 注意事项

1. 本系统为演示目的，密码采用明文存储，生产环境请使用加密存储
2. 认证采用简单的 user-id header 机制，生产环境请使用 JWT 或其他安全认证方式
3. 首次启动会自动初始化数据库和种子数据
4. 数据库文件位于 `data/childcare.db`，删除后会重新初始化
